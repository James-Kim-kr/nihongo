# 에듀테크 PRD 작업 가이드

이 디렉터리는 에듀테크 서비스를 기획·설계하기 위한 PRD(제품요구사항문서) 템플릿과 샘플을 담고 있습니다. `PRD.md`를 기준으로 작성하고, 필요 시 체크리스트와 AC(Gherkin) 예시를 참고하세요.

## 구성
- `PRD.md`: 전체 PRD 템플릿(에듀테크 특화 섹션 포함)
- `AC-examples.md`: 수용 기준(Gherkin) 예시 모음
- `Launch-Checklist.md`: 런치 전 최종 점검 목록

## 사용 방법
1) `PRD.md` 복사 또는 직접 편집으로 v1 초안 작성
2) 섹션별 [TODO] 항목을 채우며, 사용자/수업 맥락과 근거(데이터·교육학)를 함께 기록
3) 핵심 기능은 AC(수용 기준)를 동반해 명확히 정의 → `AC-examples.md` 참고
4) 릴리스 전 `Launch-Checklist.md`로 접근성/보안/안전/운영 항목 점검

## 팁
- 학습효과 증명 가능성에 집중: 학습 목표-평가-콘텐츠-분석이 일관되게 연결되어야 합니다.
- AI 기능은 모델/프롬프트/안전/로그/평가를 별도로 명시하세요.
- 공교육/아동 보호 규정, 접근성(WCAG 2.1 AA), 개인정보보호(GDPR/FERPA/COPPA)를 초기 설계에 포함하세요.
